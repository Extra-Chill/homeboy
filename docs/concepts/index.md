# Concepts

Concept docs explain Homeboy's mental model and vocabulary. Use these when you need to understand how commands fit together before looking up exact flags.

If you are trying to perform a task, start with [Workflows](../workflows/index.md). Use concepts when a workflow mentions a term such as component, runner, structured evidence, scope, or Code Factory.

## Core Concepts

- [Homeboy model](homeboy-model.md) - components, extensions, commands, evidence, runners, and operations.
- [Structured evidence](structured-evidence.md) - human output, JSON output, persisted runs, and reviewer-safe artifacts.
- [Code Factory](code-factory.md) - automated code maintenance through lint, test, audit, release, and deploy stages.
